## 3. "왜 계속 바뀌었나" — 4가지 실패 사유 (ver.md 증거)

**① 신호가 안 떠서 (최대 원인).** 복잡한 다중조건 전략이 진입 신호를 거의 못 만듦.
> 8407 ver.md: *"VictoriaDuke → TTM Squeeze … BTC/ETH/SOL 모두 신호 생성 실패"*
- VictoriaDuke = 추세선 **3접점** 돌파 → 구조적 신호 희소 (8408·8501 거래 몇 건뿐).
- KY Reversal = **4중 진입조건** → 동시 충족 드묾. → "거래를 안 하니 실패"로 보고 갈아탐.

**② 사실은 전략이 아니라 버그 (무신호의 진짜 범인).**
> 8401 ver.md: *"신호 생성 실패 근본 해결 — dropna()로 78% 데이터 손실"*, NaN, bb2_mid 인자 누락…
- 데이터 처리 버그로 신호가 막힌 걸 **전략 탓으로 오인** → 멀쩡한 전략 폐기(낭비).

**③ 백테스트 수치 추격 (과최적화).**
> 8407 ver.md: TTM 회귀 사유 = *"OOS 검증 통과, 승률 40%, PF 1.64"*
- "이 전략 백테스트가 더 좋네" → 갈아탐 → 라이브선 또 별로 → 또 갈아탐.

**④ 비현실적 목표가 강제한 무한 교체.**
> v3.3.0: *"전략 7종 최적화 — 일일 +1% 목표 구조"* (= 연 365%)
- 어떤 전략도 못 맞추니 "이것도 안 되네" → 영원히 교체.

## 4. 진단 & 처방

**진단:** 1주일에 전략 5번 교체 = 그 자체가 문제. 각 전략을 **통계적으로 의미 있는 표본(수십~수백 거래)이 쌓이기 전에** 버림 → 운/실력 분간 불가. 전략이 나빠서가 아니라 **신호 희소 + 구현 버그 + 비현실적 목표** 3박자로 "안 된다"고 착각하여 갈아타기를 반복.

**처방:**
1. 하나를 정하고 **최소 2~3주·봇당 100거래** 쌓을 때까지 안 건드리기.
2. 바꾸기 전에 **"무신호가 전략 탓인가 버그 탓인가"** 먼저 검증. (예: 2026-06-13 8401~8406 캔들동결 버그 = 전략 아닌 데이터 버그)
3. 목표를 **연 6~20%**로 현실화 → "안 된다" 착각으로 갈아타지 않게.
